Vertel eerst de context (wie / wat / voor wie)
• “Ik ben grafisch ontwerpstudent en maak een creatieve tool
met p5.js.”
• “De tool moet draaien in een apart bestand
sketch.js in een HTML-pagina.”
Zeg heel duidelijk wat je WIL en wat je NIET wil
• “Gebruik p5.js en schrijf alleen JavaScript voor sketch.js.”
• “Geen HTML, geen CSS, geen CDN-links in je antwoord.”
• “Leg alles uit met duidelijke comments, op beginnerniveau.”
Beschrijf de tool als regels, niet als ‘maak iets
cools’
(Denk in input + regels + variatie)
• Canvas: formaat, sfeer (“800×800, donker, poster-achtig”).
• Input: “reageer op muis / toetsen / random / webcam / tekst”.
• Regels: “maximaal 20 cirkels, geen overlap, 1 kleurpalet”.
• Variatie: “bij elke spatiebalk een volledig nieuwe
compositie.”
Vraag altijd om begrijpbare structuur
• “Gebruik setup() en draw().”
• “Zet bovenaan parameters (aantal vormen, kleuren, marges) als
variabelen.”
• “Commentaar bij elke belangrijke regel zodat ik het kan
aanpassen.”
Itereren & debuggen met AI
Bij fouten: plak je code + foutmelding en vraag:
• “Hier is mijn volledige sketch.js en de foutmelding. Leg stap
voor stap uit wat er mis is en verbeter de code.”
Bij uitbreiding:
• “Pas deze code aan zodat … (extra parameter, andere input,
andere regel).”

binnen dezelfde “taal”?
• Hoe ver kan je gaan in variatie zonder je identiteit te
verliezen?

• Kan de webcam geconnecteerd worden om posters te genereren?
Je bouwt geen affiche, je bouwt een systeem dat
affiches uitspuugt.

TOOL
Tekst wordt geen neutrale drager van informatie,
maar een stem met emoties. Je tool “leest” een
woord of zin en vertaalt dat naar typografische
keuzes: volume, afstand, spanning, kleur.
• Hoe ziet “angst” eruit als letterspatiëring?
• Hoe klinkt “feest” in corpsgrootte?
• Kan tekst geanimeerd worden met de webcam?
• Kunnen sliders ingezet worden om teksten te veranderen?
Je ontwerpt een kleine machine die stemming
omzet in typografie, alsof je een emotionele filter
over taal schuift.
